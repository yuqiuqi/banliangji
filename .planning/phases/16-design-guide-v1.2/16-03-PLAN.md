## Task 16-03-2 — §11.5 图表柱 Stagger 改用 `withDelay`

<read_first>
- `.planning/IOS26-DESIGN-GUIDE.md` §11.5 图表柱 Stagger Spring
- `.planning/IOS26-DESIGN-GUIDE-REVIEW.md` §四 ⚠️黄灯表「§11.5 图表柱 `setTimeout` stagger」
</read_first>

<action>
在 `.planning/IOS26-DESIGN-GUIDE.md` §11.5，把代码示例：

```tsx
useEffect(() => {
  points.forEach((pt, i) => {
    setTimeout(() => {
      barHeights[i].value = withSpring(targetH, SPRING.UI);
    }, i * 30); // stagger 30ms
  });
}, [points]);
```

替换为：

```tsx
import { withDelay, withSpring } from 'react-native-reanimated';

useEffect(() => {
  points.forEach((pt, i) => {
    const targetH = maxAmount > 0 ? (pt.amount / maxAmount) * 132 : 0;
    // Reanimated worklet 驱动，JS 线程忙时也保持节奏稳定
    barHeights[i].value = withDelay(
      i * 30,
      withSpring(Math.max(4, targetH), SPRING.UI)
    );
  });
}, [points, maxAmount]);
```

**并在代码下方加说明段：**

> **为什么不用 `setTimeout`：** 🛠️ RN-T1 📦 react-native-reanimated
> - `setTimeout` 在 JS 线程忙（如滚动 + 数据刷新同时发生）时会漂移，stagger 节奏破坏
> - `withDelay(ms, withSpring(...))` 是 Reanimated worklet，在 UI 线程执行，节奏恒定
> - 额外好处：组件卸载时 Reanimated 自动取消动画，无需手动 `clearTimeout`
</action>
