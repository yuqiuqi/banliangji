## 1. 问题陈述（与系统差距）

当前 RN + `expo-blur` 实现的是 **传统实时模糊（近似 UIBlurEffect）**：高斯/噪声型散射，**缺少** 设计叙述中 **Liquid Glass** 强调的 **Lensing（折光/聚光）** 与 **随交互瞬时「充能」的凝胶态形变**。因此会出现：

| 维度 | Liquid Glass 取向（归纳） | 常见 RN 实现 | 感知差距 |
|------|---------------------------|--------------|----------|
| **光学** | 动态弯曲、塑造、**集中**光线；与「散射型」模糊区分 | `BlurView` + 半透明叠色 | 偏「雾面」，缺少**透镜感**与边缘折射暗示 |
| **纵深** | 多层材质协同；大尺寸浮层时**更厚**、阴影更强 | 单层 blur + 固定 shadow | **立体感弱**，Tab/Sheet 与内容层分离不明显 |
| **透明度** | 各层随背后内容**连续自适应**（对比度、着色、分离阴影） | 固定 `tint` / `intensity` | 滚动或内容变化时**分离度与可读性**偏弱 |
| **动效** | **视觉与动效一体**；按压即时 flex、状态迁移时材质**形变/融合** | 线性 opacity 或简单 scale | 缺**流体感**与「控件抬升为玻璃」的叙事 |

> 结论：**不是「再调大一点 blur」能解决的**，需要在 **分层策略 + 动效曲线 + 降级语义** 上重新设计；RN 侧以 **可验收的近似 + 文档化偏差** 为目标。

## 2. 设计原则（归纳）

1. **Lensing 作为识别特征** — 用「光的弯折」传达**存在、运动与形态**，在让内容透出的同时保持与背景的分离。
2. **显隐过渡** — 避免简单淡入淡出；通过 **逐步调制** 材质强度保持观感一致。
3. **动效与材质同源** — 动效应 **顺滑、响应快**；交互时材质可 **瞬时收紧/提亮**。
4. **按压抬升** — 控件可在交互时**临时进入更显著的玻璃态**，静止时保持视觉安静。
5. **上下文形变** — 状态切换时避免「硬切」导航层，尽量 **同一浮动层上连贯变化**。
6. **多层与自适应** — 随背后内容变化调整 tint、对比与**分离用阴影**（如列表滚动时顶栏分离感）。
7. **尺度与厚度** — 浮层变大时 **阴影与不透明度略增**，模拟「更厚」的层次（用 **分段参数** 近似）。

## 3. React Native / Expo 技术现实（仅本地实现）

| 能力 | 现状 | 对 Phase 11 的含义 |
|------|------|---------------------|
| `expo-blur` | iOS 原生视觉模糊视图；**非**系统私有 Liquid Glass 着色器 | **Tier-1 底座**；用 **叠层**（细线高光、阴影、渐变 mask）增强边缘与层次 |
| Reanimated | 布局/手势线程动画 | **Tier-1**：按压 spring、Tab 转场、Sheet 与列表联动 |
| 自定义原生视图 | 可封装未来 SDK | **Tier-2 可选**；本阶段不强制、**不涉及网络** |
| Android | Blur 与 iOS 不一致 | **实色近似 + 文档标注**；不与 iOS 强行像素一致 |

**实现注意：** `BlurView` 与列表层序、内容更新后 blur 刷新等，须在 Phase 11 **验收清单**中逐屏确认（仅本地跑 App，无需联网）。

## 4. 专业落地方案（分层）

### 4.1 材质栈（建议）

对 **Tab 栏、顶栏、大浮层（记一笔 Dock、Sheet 把手区）** 采用统一「栈」，按 elevation 选参数：

```
[ 内容 ]
  ↑
[ 可选：极薄分隔阴影层 — Scroll 时增强对比 ]
  ↑
[ BlurView — 项目约定 light/dark 枚举 ]
  ↑
[ 半透明 tint — 随 ColorScheme 与 iosSemantic 绑定 ]
  ↑
[ 1px hairline 顶部高光（线性渐变）]
  ↑
[ 圆角裁切 + 外阴影 — 大浮层用较大 radius + 较深 shadowOpacity ]
```

- **小控件**：blur 可减弱或 **实色 + 按压高光**。
- **大浮层**：提高 **shadowRadius / opacity** 与 **tint**，模拟「变厚」。

### 4.2 动效栈（建议）

| 场景 | 推荐 | 参数方向 |
|------|------|----------|
| 按压 | `Pressable` + Reanimated `withSpring` | 轻微 scale（0.97–0.99） |
| Tab 切换 | spring；图标轻微位移 | 避免单一线性 `timing` |
| 导航转场 | 与 Phase 9 栈顶阴影 **连贯** | 避免 Tab 与卡片 **脱节** |
| Sheet / 记一笔 Dock | spring + 遮罩非线性进入 | 避免硬弹 |

**曲线：** 优先 **spring**；避免把 Material 默认曲线直接套在玻璃控件上。

### 4.3 透明度与可读性（与 A11Y 衔接）

- **降低透明度**：统一封装检测，Blur 切换为 **不透明底 + 分隔线与字色**（**A11Y-01**）。
- **深色模式**：tint 与 `iosSemantic` **联动**（**THEME-01**）。

### 4.4 验收（建议写入 Phase 11 VERIFICATION）

1. **静态：** Tab、记一笔 Dock、主列表顶栏 — 边缘层次非「一块灰雾」。
2. **滚动：** 快速滚动时顶栏/Tab **仍可读**。
3. **动效：** 按压、Tab、记一笔 — **无主导性线性僵硬感**。
4. **辅助功能：** 降低透明度开/关各跑主路径。

## 5. 风险与诚实边界

- **无法 1:1 复刻** 系统级 Liquid Glass；本方案为 **本机工程近似**。
- 若将来 SDK 提供可嵌入 RN 的公开材质 API，**单点替换** `BlurView` 封装即可。
- 性能：多层 blur + Reanimated 须 **真机**测帧；必要时 **减少层数**（仍仅本地，无网络）。
