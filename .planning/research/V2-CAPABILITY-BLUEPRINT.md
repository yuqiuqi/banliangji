# 半两记 v2 能力蓝图（内部归档）

**用途：** 记录 **半两记** 在 v2 周期希望具备的 **模块级能力** 与工程落点，供 `REF-*` 需求与 Phase 5–7 追溯。  
**Date:** 2026-04-21  

> 本文档描述的是 **本产品** 的能力切片，不指向任何外部成品 App 的「对照」或「迁移」。

## 模块能力矩阵

| 领域 | 半两记产品能力 | 工程落点 |
|------|----------------|----------|
| 记账主路径 | 记支出 / 分类 / 详情 / 编辑闭环 | `CreateBill` / `BillDetail` 等信息密度与动效 |
| 明细与日期 | 按日浏览与筛选账单 | 日历与明细列表联动 |
| **预算** | 独立「预算」能力 | 预算数据模型 + UI（月度/分类等） |
| **资产** | 资产/账户余额与快照 | SQLite 表扩展 + 资产相关屏 |
| 图表 | 可读、可切换周期的支出视图 | Phase 3 数据语义 + 表现层迭代 |
| 登录/云账号 | 多用户云体系 | **默认不纳入 v2 核心**（与离线单机 Core Value 冲突）；若要做单独立项 |
| 发现 / 通讯录 / 在线 API 聚合 | 运营向能力 | **默认 Out of Scope**；另里程碑 |

## 技术约束

- 交付栈保持 **Expo / React Native / TypeScript**；不引入需额外原生 GUI 绑定或混合语言 UI 层的依赖。  
- 与 `PROJECT.md` **Core Value** 冲突的能力须单独 REQ + Out of Scope 说明。
